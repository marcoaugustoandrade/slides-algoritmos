Procedimentos

É um conjunto de instruções que executa determinada tarefa e não possui nenhum retorno. (PUGA & RISSETI, 2016).

Também é conhecido como sub-rotina.

Declaração de um procedimento


	funcao nome_do_procedimento(parâmetros)
	{
		//Execução do procedimento
	}

Declaração de um procedimento


	funcao nome_completo(cadeia nome, cadeia sobrenome)
	{
		escreva(nome + " " + sobrenome)
	}

Utilizando um procedimento


	funcao inicio()
	{
		nome_completo("José", "Oliveira")
	}

Funções

É um conjunto de instruções que executa determinada tarefa e retorna algum valor. (PUGA & RISSETI, 2016).

O tipo de retorno pode ser: inteiro, real, caracter, cadeia, logico.
O retorno de uma função pode ser atribuído a uma variável.

Declaração de uma função


	funcao tipo_de_retorno nome_da_funcao (parâmetros)
	{
		//Execução da função e retorno
	}

Declaração de uma função


	funcao real media(real n1, real n2)
	{
		retorne (n1 + n2) / 2
	}

Utilizando uma função


	funcao inicio()
	{
		escreva(media(13.0, 7.0))
	}

Escopo de variáveis

  • Uma variável pode ser local ou global.
  • As variáveis locais podem ser acessadas somente na função/procedimento que estão declaradas.
  • As variáveis globais podem ser acessadas em qualquer local do algoritmo.

Escopo de variáveis


	real media = 0.0

	funcao calcular_media_aritmetica(real num1, real num2)
	{
		media = (num1 + num2) / 2
	}
	funcao calcular_media_ponderada(real num1, real num2)
	{
		media = (num1 * 0.4 + num2 * 0.6) / 2
	}
	funcao inicio ()
	{
		caracter opcao
		escreva("Selecione a opção: ")
		leia(opcao)
		se (opcao == 'a')
			calcular_media_aritmetica(7.0, 9.0)
		senao se (opcao == 'p')
			calcular_media_ponderada(7.0, 9.0)
		escreva(media)
	}

Passagem de parâmetros

  • Pode ser por valor ou por referência.
  • Na passagem por valor os valores informados como parâmetros são copiados para dentro do módulo, criando-se assim uma nova variável local com o conteúdo.
  • Na passagem por referência os valores informados são acessados, independente de onde estejam, e assim qualquer mudança no módulo é refletida.

Passagem de parâmetros por valor


	funcao real calcular_media (real num1, real num2)
	{
		retorne (num1 + num2) / 2
	}
	funcao inicio()
	{
		real num1 = 10.0, num2 = 8.0
		escreva(calcular_media(num1, num2))
		escreva(calcular_media(7.0, 5.0))
	}
São criadas 2 variáveis locais na função calcular_media: num1 e num2, do tipo real, e os valores passados como parâmetros são copiados para dentro dessas variáveis.

Passagem de parâmetros por referência


	funcao real calcular_media (real &num1, real &num2)
	{
		num1 += 1
		retorne (num1 + num2) / 2
	}
	funcao inicio()
	{
		real num1 = 10.0, num2 = 8.0
		escreva(calcular_media(num1, num2))
		escreva(num1)
		escreva(num2)
	}
As 2 variáveis locais da função inicio são acessadas pela função calcular_media.
